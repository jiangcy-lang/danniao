# 海马体规范：动态认知树与信息触发沉淀

> 本文档**取代**旧版 `01-基础架构与数据沉淀规范.docx` 中「一次性预置 8 个扁平节点」的做法。

## 1. 核心原则

1. **废弃扁平结构**：记忆为树状层级；初始不预置颜色/大小/味道等子节点。  
2. **渐进式生长**：仅告知「苹果」→ 至多一个孤立主干；高信息量交互才繁衍子节点。  
3. **信息门控**：禁止对任意输入直接无脑 INSERT；须经匹配 → 误差评估 → 条件繁衍。  
4. **数据沉淀**：权重可稀释，边/节点不物理删除。

## 2. 技术栈

- Python 3.9+  
- NetworkX `DiGraph`（内存认知树）  
- JSON 持久化（`nx.node_link_data`）  
- 特征抽取 MVP：规则词典（可替换，门控接口不变）

## 3. 节点与边

### 主干节点（trunk）

| 字段 | 说明 |
|------|------|
| `node_id` / `concept` | 概念名，如「苹果」 |
| `kind` | `"trunk"` |
| `access_weight` | 访问/激活强度，常规提及时提升 |
| `creation_time` | ISO 8601 |

### 子节点（dimension-feature）

| 字段 | 说明 |
|------|------|
| `concept` | 命名：`{维度}-{特征值}`，如 `颜色-红`、`味道-甜` |
| `kind` | `"feature"` |
| `dimension` | 如 `颜色` |
| `value` | 如 `红` |

### 边

`trunk → feature`，属性含 `weight`（默认 0.1）、`dimension`、`last_activated_time`。

## 4. 信息触发沉淀门控

```text
parse(text) → trunk, features[]

若无法识别主干 → no_op

若主干不存在：
  创建孤立主干
  若同句已含 features → 允许同次繁衍（MVP）
  否则结束（仅孤立节点）

若 features 为空：
  仅激活主干，提升 access_weight，不繁衍

若 features 非空：
  new = 主干下尚不存在的 (维度, 值)
  若 new 为空 → 激活已有匹配子节点（强化），不繁衍
  若 new 非空 → 标记预测误差，动态生成子节点并建边
```

### 与好奇心的关系（阶段划分）

| 阶段 | 行为 |
|------|------|
| **本轮 MVP** | 输入本身已携带新维度特征（如「红色的甜苹果」）→ 直接沉淀繁衍 |
| **后续** | 仅有冲突线索时先升预测误差 → 探索/询问 → 高信息答复到达后再繁衍 |

## 5. 最小输出（海马体侧）

- `get_node` / `get_children` / `describe(trunk)`  
- `describe` 须能用自然语言或结构化摘要说明：当前主干、子节点列表（表达「我知道什么」）

## 6. 验收标准

1. 空树 + 输入「苹果」→ 仅 1 个主干「苹果」，无出边。  
2. 再输入「红色的甜苹果」→ 在「苹果」下生成 `颜色-红`、`味道-甜`。  
3. 再输入「苹果」→ 子节点数量不变（常规输入不繁衍）。  
4. 调用 `describe("苹果")` → 输出包含主干及已有特征子节点（证明具备最小表达）。

## 7. 明确禁止

- 初始化时预置红/绿/甜/酸/大/小等扁平概念全集  
- 常规提及主干时自动幻想出子属性  
- 将搜索/交付逻辑写入海马体包内部
