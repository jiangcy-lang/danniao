# 海马体规范：向量认知空间与信息触发沉淀

> 隶属 **[00-master-spec.md](./00-master-spec.md)** Step 1–7。  
> 取代旧 `01-基础架构…docx` 的扁平 8 节点初始化。  
> 向量节点重构 + 分层存储详见 **[danniao-vector-node-redesign/](../danniao-vector-node-redesign/danniao-vector-node-redesign.html)**。

## 1. 核心原则

1. 向量即节点：节点本体 = 向量；标签是投影（2026-07-22 重构）
2. 树状层级，空图起步，禁止预置子属性
3. 渐进式生长：仅「苹果」→ 孤立主干；高信息交互才繁衍
4. 信息门控：匹配 → 误差评估 → 条件繁衍（非无脑 INSERT）
5. 统一向量-图空间：向量视图（ChromaDB）+ 图视图（NetworkX），同一认知空间的两个视图
6. 分层存储：原始日志（全留存）→ 活跃向量（工作记忆）→ 归档向量（淡化可唤起）
7. 数据沉淀：权重可衰减休眠，迁出至归档层，**节点/边不物理删除**
8. 知识可继承但必须经认知机制消化：继承知识作为输入灌入门控系统，由预测误差自动分拣——低误差吸收、中误差繁衍、高误差质疑（2026-07-22 新增）

## 2. 节点与边

### 向量节点（VectorNode）

| 字段 | 说明 |
|------|------|
| `embedding` | 节点身份 —— 语义空间中的位置（Step 4A 接入） |
| `node_id` | `hash(embedding)` —— 向量相同则 ID 相同 |
| `label` | 人类可读投影（可选），如「苹果」 |
| `kind` | `trunk` / `feature` / `abstraction` |
| `modality` | 来源模态：text / image / multi |
| `activation` | 当前激活水平 |
| `creation_time` | ISO 8601 |

### 树结构 = 向量层次化特化

子节点是父节点在某个方向上的偏移向量。「青苹果」是靠近「苹果」但在颜色方向上有偏移的向量。

### 边

`trunk → feature`：`weight`、`dimension`、`last_activated_time`

显式边 = 学到的强化关系；向量邻近 = 天然的隐含关系。两者叠加。

## 3. 分层存储

| 层级 | 技术 | 内容 | 职责 |
|------|------|------|------|
| **第一层：原始日志** | SQLite 追加写 | 每次交互完整原文 | 情景记忆，永不删除 |
| **第二层：活跃向量** | ChromaDB 内存 + NetworkX | 高权重/近期活跃节点 + 图拓扑 | 工作记忆，门控/扩散/表达仅此层操作 |
| **第三层：归档向量** | ChromaDB 磁盘 + SQLite 图索引 | 权重衰减至阈值以下 | 长期记忆，淡化可唤起 |

层间流转：写入（活跃+日志）→ 迁出（淡化）→ 唤起（重新激活）→ 巩固（压缩重组）。

**关键约束**：活跃层始终保持在 NetworkX 可控范围内（数千节点）。

## 4. 信息触发沉淀门控

```text
embed(text) → vector                 # Step 4A：嵌入管道
vector match → trunk                  # 余弦相似度 > 阈值

主干不存在 → 仅创建孤立向量节点（无 features 时）

features 为空 → 仅 activation 增强，不繁衍

features 非空且为新维度 → 预测误差 → 节点繁衍
（Step 4B：预测误差从布尔变为连续值 —— 余弦距离）
```

Step 3 MVP 验收用例（文本规则词典）：

- 「苹果」→ 无子节点
- 「青苹果」→ `颜色-青`
- 「被咬了一口的青苹果」→ `颜色-青`、`形状-缺口`（扩展用例）

Step 4A 验收用例（向量匹配）：

- 「苹果」+「apple」→ 匹配同一向量节点
- 「苹果」图 +「苹果」文本 → 匹配同一向量节点（Step 7 多模态）

### 4.1 门控的传承分拣机制（2026-07-22 新增）

预测误差门控天然支持知识继承的分拣，无需额外设计：

| 误差水平 | 门控行为 | 对应认知模式 | 占比 |
|----------|----------|------------|------|
| **低误差** | 权重增强，不繁衍 | 吸收既定事实（如 1+1=2） | ~95% |
| **中误差** | 局部繁衍，扩展维度 | 学习新概念领域 | ~4% |
| **高误差** | 触发探索，验证或推翻 | 质疑既定事实 | ~1% |

**传承输入处理**：继承的知识（巩固后的教材）作为模拟输入序列灌入门控系统，门控自动按误差水平分拣。大部分既定知识快速吸收，少量扩展认知维度，极少量触发质疑——与人类"读大学"模式一致。

**禁止**：绕过门控直接写入认知树数据（跳过了认知机制消化，导致动力学与权重不匹配）。

## 5. 扩散激活（Step 4B）

激活沿图边（强，受边权重调制）和向量邻近（弱，受余弦相似度调制）双向传播。

## 6. 输出

- `print_tree(trunk)` — 打印树状结构（Step 1 验收）
- `describe(trunk)` — 模板拼接描述当前认知（Step 1–5）
- `ExpressionEngine` — 激活模式 → 自然语言（Step 6，LLM 仅做解码器）

## 8. 禁止

- 初始化预置红/绿/甜/酸等全集
- 常规提及主干时自动幻想子属性
- 在海马体包内实现搜索/交付
- LLM 存储记忆或做决策（仅做语言解码器）
- 活跃层无限膨胀（必须通过迁出机制控制规模）
- 绕过门控直接写入认知树数据作为"传承"（继承知识必须经子代门控消化）
