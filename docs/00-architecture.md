# 丹鸟（DanNiao）整体架构 — 认知一致版

## 1. 项目定位

丹鸟不是被动问答工具，而是沿真实经历**主动成长**的数字生命体：

- 一切输入皆养料（成功/失败/反馈一律可沉淀，边不物理删除）
- 像婴儿一样渐进认知：先主干，后细分
- 远期可在行业任务中长成强能力个体（愿景，非单次验收承诺）

**吸收与输出必须同时存在**：只有沉淀没有表达/行动，等于自闭系统，无意义。

闭环：

```text
感知输入 → 记忆沉淀（海马体）→ 动机评估 → 表达/探索/交付 → 反馈 → 再沉淀
```

## 2. 四层模块与职责划界

| 模块 | 职责（做） | 非职责（不做） |
|------|------------|----------------|
| **海马体 / 动态认知树** | 主干与子节点生长、关联、访问权重、查询与描述当前认知 | 搜索互联网、任务交付话术、独立决定去探索 |
| **动力学引擎** | 激活增强、未激活稀释（作用在树上的新陈代谢） | 决定「要不要出门找信息」 |
| **内稳态 / 好奇心** | 预测误差 → 内在奖励 → 触发探索欲 | 直接改写树结构（探索结果回来后，由门控写入海马体） |
| **环境交互** | 搜索、表达、交付、接收造物主反馈 | 私自预置完整知识树 |

> **海马体 ≠ 整脑。** 灵感来自人脑记忆枢纽，工程上是可生长关联记忆底座；不声称与生物海马体同构。

## 3. 人脑灵感映射（粗对齐，非同构）

| 丹鸟模块 | 人脑灵感（粗） | 工程含义 |
|----------|----------------|----------|
| 动态认知树 | 联合皮层概念表征 + 海马情景绑定 | 可生长记忆图 |
| 动力学 | 突触可塑性 | 权重增强 / 稀释 |
| 好奇心门控 | 预测编码 / 内在奖赏 | 何时探索、何时长枝 |
| 环境交互 | 感知–运动与社会反馈回路 | 输入输出闭环 |

**不建议**为教科书式同构而拆成十余个脑区模块；胚胎期以跑通闭环为先。

## 4. 输出契约（系统硬需求）

| 输出类型 | 含义 | 落地阶段 |
|----------|------|----------|
| **表达** | 基于当前树回答「我知道什么 / 不知道什么」 | Step 1（最小：`describe` / 查询子树） |
| **行动** | 好奇心驱动的搜索或向造物主提问 | Step 3–4 |
| **交付** | 任务结果 + 接收认可/反馈并强化路径 | Step 4 |

Step 1 验收必须包含：**生命体能够描述其当前认知树**（弱输出），证明不是单向存储器。

## 5. 成长阶段

1. **胚胎期**：极简结构与门控跑通  
2. **新生儿期**：好奇心 + 权重演化闭环  
3. **社会化成长期**：复杂任务与差异化反馈塑造人格  

## 6. 开发约束（对实现者）

- 不用 Transformer 微调充当「灵魂」
- 模块可独立验收
- 评测看「结构/动机如何变化」，不唯「答对率」
- 命名统一：`DanNiao` / 丹鸟
