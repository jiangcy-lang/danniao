## 1. 项目定位

丹鸟是沿真实经历**主动成长**的数字生命体：一切输入皆养料，知识可继承但必须经认知机制消化；像婴儿一样先主干后细分，也可像大学生一样批量吸收既定知识；**吸收与输出必须同时存在**。

```text
感知输入 → 嵌入管道 → 统一向量-图空间（分层存储）→ 动力学 → 预测误差门控
         → 扩散激活 → 内稳态驱动力 → 表达 / 探索 / 交付 → 内外奖励 → 再沉淀
```

## 2. 模块与职责划界

| 模块 | 职责 | 非职责 |
|------|------|--------|
| **海马体 / 向量认知空间** | 向量节点、统一向量-图空间、分层存储、门控写入、查询与表达 | 搜索、交付、独立决定去探索 |
| **感知管道** | 文本/图像 → 向量嵌入 | 存储记忆、做决策 |
| **扩散激活** | 激活沿图边 + 向量邻近传播 | 改写记忆结构 |
| **动力学引擎** | Hebbian 增强、全局衰减 λ | 决定是否探索 |
| **内稳态 / 动机** | 预测误差 → 好奇心 / 置信度 / 驱动力判断 | 直接改树（结果经门控写入） |
| **探索行动** | 好奇心驱动搜索 / 提问 / 结果回注 | 决定记忆结构 |
| **表达引擎** | 激活模式 → 自然语言（LLM 仅做解码器） | 存储记忆、做推理 |
| **记忆巩固** | 重放 / 强化 / 合并 / 抽象 / 层间流转 | 主动探索 |
| **代际传承** | 父代巩固知识 → 教材 → 子代门控消化 → 出厂 | 直接复制认知树数据 |
| **外骨骼接口** | 挂载工作环境、任务交付、环境反馈 | 存储记忆、做决策 |
| **环境交互** | 搜索、表达、交付、造物主反馈 | 预置完整知识树 |

> 海马体 ≠ 整脑；人脑映射为灵感粗对齐，不要求生物同构。

## 3. 分层存储架构

> 造物主指示：人类几乎所有记忆都不会被真正删除，而是淡化，存储是海量的。

| 层级 | 类比 | 技术 | 规模 | 职责 |
|------|------|------|------|------|
| **第一层：原始日志** | 情景记忆 | SQLite 追加写 | 百万条 ~1-2GB | 每次交互完整原文，永不删除 |
| **第二层：活跃向量** | 工作记忆 | ChromaDB 内存 + NetworkX | 数百~数千节点 | 高权重/近期活跃节点，扩散激活/门控/表达仅此层操作 |
| **第三层：归档向量** | 长期记忆 | ChromaDB 磁盘 + SQLite 图索引 | 百万级向量 | 权重衰减至阈值以下，淡化但不消失，高相似可唤起 |

层间流转：写入 → 迁出（淡化）→ 唤起（重新激活）→ 巩固（压缩重组）。

**关键约束**：活跃层始终保持在 NetworkX 可控范围内。内存占用不随认知总量增长。

## 5. 输出契约

| 类型 | Step |
|------|------|
| 表达（describe / 打印树） | 1 |
| 行动（搜索 / 提问） | 5–6 |
| 交付 + 社会性奖励 | 6 |
| LLM 表达（激活模式→自然语言） | 6 |
| 巩固教材（供传承） | 7 |
| 挂载外骨骼 + 任务交付 | 8+ |

## 6. 成长阶段

1. 胚胎期（Step 1–3）：结构 + 门控 + 动力学跑通
2. 向量化期（Step 4A–4B）：向量节点 + 嵌入管道 + 扩散激活
3. 新生儿期（Step 5–6）：内稳态 + 持续心智 + 探索 + 表达 + 闭环
4. 社会化成长期（Step 7+）：记忆巩固 + 多模态 + 差异化任务塑造人格
5. 传承期（Step 8+）：父代编教材 → 子代"读大学"毕业 → 挂载外骨骼参与工作 → 在职持续微调
